       Lsof修订版4.91列出了其标准输出文件信息
       关于由以下UNIX方言的进程打开的文件：

            Apple Darwin 9和Mac OS X 10. [567]
            对于基于AMD64的系统，FreeBSD 8 [234]，9.0和1 [012] .0
            适用于基于x86的系统的Linux 2.1.72及更高版本
            Solaris 9,10和11

       （有关信息，请参阅本手册页的“ 分发”部分
       如何获得最新的lsof修订版。）

       打开的文件可以是常规文件，目录，块特殊
       文件，字符特殊文件，执行文本引用，a
       库，流或网络文件（Internet套接字，NFS文件或
       UNIX域套接字。）特定文件或文件中的所有文件
       系统可以通过路径选择。

       而不是格式化的显示，lsof将产生可以的输出
       由其他程序解析。有关详细信息，请参阅-F，选项说明和其他程序的
        输出部分。

       除了生成单个输出列表外，还会运行lsof
       重复模式。在重复模式下，它将产生输出，延迟
       重复输出操作，直到中断或退出停止
       信号。有关详细信息，请参阅+ | -r [t [m <fmt>]]选项说明
       信息。

安全         顶部
       Lsof有三个可能引起安全问题的功能。首先，它
       默认编译模式允许任何人列出所有打开的文件
       它。其次，默认情况下，它创建用户可读和用户可写
       设备缓存文件在真实用户ID的主目录中
       执行lsof。（list-all-open-files和设备缓存功能编译lsof 
       时可能会被禁用。）第三，它的-k和-m选项
       命名备用内核名称列表或内存文件。

       限制所有打开文件的列表由。控制
       编译时HASSECURITY和HASNOSOCKSECURITY选项。什么时候
       定义了HASSECURITY，lsof将只允许root用户列出
       所有打开的文件。非root用户可能只列出打开的文件
       具有与真实用户相同的用户标识号的进程lsof进程的
       ID号（用户登录的进程）。

       但是，如果定义了HASSECURITY和HASNOSOCKSECURITY，
       任何人都可以列出打开的套接字文件，只要它们被选中
       在-i选项。

       如果未定义HASSECURITY，则任何人都可以列出所有打开的文件。

       帮助输出，响应-h或- ？  选项，给出
       HASSECURITY和HASNOSOCKSECURITY定义的状态。

       见保安的第00README的文件lsof的 
       信息分发建设lsof的与HASSECURITY
       和HASNOSOCKSECURITY选项已启用。

       它的路径是如何形成的。出于安全考虑，它是
       重要的是要注意，在默认的lsof分布中，如果真实的话执行lsof的

       所述-D选项提供有限的手段，用于指定所述设备的缓存
       文件路径。它呢？  函数将报告只读和写入lsof将使用的
       设备缓存文件路径。

       当-D b，r和u函数可用时，您可以使用它们
       请求在特定位置构建缓存文件
       （b [ 路径 ]）; 读但不重建（r [ 路径 ]）; 或阅读和重建
       （你 [ 路径 ]）。在b，[R ，和ü功能在某些限制
       条件。它们在lsof过程中受到限制
       setuid的根。r函数指定的路径始终为
       只读，即使它可用。

       的b，- [R ，和ü功能也受到限制时lsof的过程
       运行setgid并且lsof不会放弃setgid权限。（看到
       在LSOF权限影响设备的缓存文件访问节
       通常不会放弃其setgid的实现列表
       允许。）

       另外一个-D函数i（用于忽略）始终可用。

       可用时，b函数告诉lsof读取设备信息
       从内核使用stat（2）函数并构建设备缓存
       文件在指定的路径上。

       可用时，r函数告诉lsof读取设备缓存
       文件，但不更新它。当路径参数伴随-Dr时，它
       命名设备缓存文件路径。在[R功能始终可用
       何时指定没有路径名参数。如果lsof不是
       运行setuid-root并放弃其setgid权限，即路径名
       参数可以伴随r函数。

       如果可用，u函数告诉lsof尝试读取和使用
       设备缓存文件。如果它无法读取文件，或者它找不到
       文件内容不正确或过时，会读取信息
       从内核，并尝试编写更新版本的
       设备缓存文件，但仅限于它认为合法的路径
       lsof进程有效且真实的UID。

DIAGNOSTICS         顶部
       使用标准错误文件上的消息标识错误。

       如果检测到任何错误，Lsof返回一（1），包括
       无法找到命令名称，文件名，Internet地址或
       文件，登录名，NFS文件，PID，PGID或要求的UID
       名单。如果指定了-V选项，则lsof将指示搜索
       它未能列出的项目。

       如果没有检测到错误并且能够检测到错误，则返回零（0）
       列出有关所有指定搜索参数的一些信息。

       当lsof无法打开访问/ dev（或/ devices）或其中一个
       子目录，或者获得文件的信息在他们的stat（2） ，它
       发出警告信息并继续。那个lsof会发出警告指示
       有关/ dev（或/ devices）中不可访问文件的消息
       在其帮助输出中 - 请求-h或> B  - ？选项 - 用
       消息：

            启用了无法访问/ dev警告。

       可以使用-w选项抑制警告消息。有可能
       当lsof是时，系统管理员也已经压制了它
       由WARNDEVACCESS定义的设置编译。在这
       例如，帮助选项的输出将包含以下消息：

            无法访问/ dev警告被禁用。

示例         顶部
       有关更全面的示例，请参阅lsof发行版的
        00QUICKSTART文件。

       要列出所有打开的文件，请使用：

              lsof的

       要列出所有打开的Internet，x.25（HP-UX）和UNIX域文件，请使用：

              lsof -i -U

       列出其PID的进程正在使用的所有打开的IPv4网络文件
       是1234，使用：

              lsof -i 4 -a -p 1234

       假设UNIX方言支持IPv6，则仅列出开放IPv6
       网络文件，使用：

              lsof -i 6

       在端口513,514或515上使用任何协议列出所有文件
       主持人wonderland.cc.purdue.edu，使用：

              lsof -i @ wonderland.cc.purdue.edu:513-515

       在任何端口上列出使用任何协议的所有文件
       mace.cc.purdue.edu（cc.purdue.edu是默认域名），使用：

              lsof -i @mace

       列出登录名“abe”或用户ID 1234的所有打开文件，或
       过程456，或过程123，或过程789，使用：

              lsof -p 456,123,789 -u 1234，abe

       要列出设备/ dev / hd4上的所有打开文件，请使用：

              lsof / dev / hd4

       要查找打开/ u / abe / foo的进程，请使用：

              lsof / u / abe / foo

       要将SIGHUP发送到打开/ u / abe / bar的进程，请使用：

              kill -HUP`lsof -t / u / abe / bar`

       要查找任何打开的文件，包括打开的UNIX域套接字文件，
       使用名称/ dev / log，使用：

              lsof / dev / log

       在名为的NFS文件系统上查找具有打开文件的进程
       / nfs / mount /指向其服务器不可访问，并假设您的服务器
       mount表提供/ nfs / mount / point的设备号，使用：

              lsof -b / nfs / mount / point

       要在禁用警告消息的情况下执行上述搜索，请使用：

              lsof -bw / nfs / mount / point

       要忽略设备缓存文件，请使用：

              lsof -Di

       获取每个进程的文件的PID和命令名称字段输出
       每个文件的描述符，文件设备号和文件inode号
       每个过程，使用：

              lsof -FpcfDi

       列出运行该进程的每个进程的描述符1和3的文件
       每隔10秒登录ID“abe”的lsof命令，使用：

              lsof -c lsof -a -d 1 -d 3 -u abe -r10

       列出运行命令的进程的当前工作目录
       这正好是四个字符长，并且有'o'或'O'
       字符三，使用-c c选项的正则表达式形式：

              lsof -c /^..o.$/i -a -d cwd

       通过关联的数字查找IP版本4套接字文件
       点式地址，用途：

              lsof -i@128.210.15.17

       查找IP版本6套接字文件（UNIX方言支持时）
       IPv6）通过其关联的数字冒号形式地址，使用：

              lsof -i @ [0：1：2：3：4：5：6：7]

       查找IP版本6套接字文件（UNIX方言支持时）
       IPv6）由一个相关的数字冒号形式地址运行
       其中的零 - 例如，环回地址 - 使用：

              lsof -i @ [:: 1]

       要获得包含当前时间的重复模式标记线，
       使用：

              lsof -rm ====％T ====

       要向前一个标记行添加空格，请使用：

              lsof -r“m ====％T ====”
BUGS         顶部
       由于lsof在搜索打开文件时读取内核内存，因此速度很快
       内核内存的变化可能会产生不可预测的结果。

       当文件有多个记录锁时，锁状态字符
       （在文件描述符之后）来自第一个测试
       锁定结构，不是来自单个记录的任何组合
       可能由多个锁结构描述的锁。

       除非使用root set-UID权限安装，否则Lsof无法按名称搜索具有限制访问权限的文件
        。除此以外
       它仅限于搜索其用户或其set-GID所在的文件
       组（如果有）具有访问权限。

       显示原始套接字的目标地址（例如，用于
       ping）取决于UNIX操作系统。有些方言存储了
       一些原始套接字协议控制块中的目标地址
       不要。

       Lsof不能总是以相同的方式表示Solaris设备号
       该LS（1）一样。例如，主设备号和次设备号
       该LSTAT（2）和定子（2）的功能的目录报告
       安装的CD-ROM文件（通常为/ cdrom）与之不同
       它为CD-ROM文件所在的设备报告的那些
       已安装（通常为/ dev / sr0）。（Lsof报告目录号码。）

       对/ proc文件系统的支持仅适用于BSD和
       Tru64 UNIX方言，Linux和从SYSV R4派生的方言 - 例如，
       FreeBSD，NetBSD，OpenBSD，Solaris，UnixWare。

       一些/ proc文件项 - 设备号，inode号和文件大小 -
       在某些方言中不可用。在/ proc中搜索文件
       文件系统可能要求指定完整路径名。

       没有为Linux进程显示文本（txt）文件描述符。
       除当前工作目录以外的文件的所有条目
       根目录和数字文件描述符标记为mem
       描述。

       Lsof无法按名称搜索Tru64 UNIX命名管道，因为它们的名称
       lstat（2）的内核实现返回一个不正确的设备号
       用于命名管道。

       Lsof无法在HP-UX 9.01,10.20和11.00上完全或正确报告
       因为对内核数据的访问权限不足或者错误导致锁定
       内核数据。请参阅lsof常见问题解答（常见问题解答部分给出了它的位置。）
       详情。

       AIX SMT文件类型是一种制作。它弥补了文件
       类型（15）未在AIX中定义的结构
       /usr/include/sys/file.h头文件。创建此类文件的一种方法
       结构是在DISPLAY变量设置为的情况下运行X客户端
       ``：0.0 ''。

       在+ | -f [cfgGn]选项没有根据基于PROC / Linux支持lsof的，
       因为它不从内核内存中读取内核结构。
